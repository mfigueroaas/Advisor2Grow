- **Archivo:** globals.css

- **` :root ` — Tokens base:** variables globales que definen los colores reales.
  - **`--page`**: color principal de la página (fondo global). Cambiarlo suele afectar el `background` de `html`/`body`.
  - **`--page-foreground`**: color del texto sobre `--page` (texto principal cuando el fondo es `--page`).
  - **`--surface`**: color de superficies como tarjetas, modales y popovers (fondo de "paneles").
  - **`--surface-foreground`**: color del texto dentro de las superficies (`--surface`).
  - **`--surface-soft` / `--surface-soft-foreground`**: superficies y su texto en variantes suaves (ej. badges, fondos secundarios).
  - **`--surface-strong` / `--surface-strong-foreground`**: superficies más contrastadas (ej. elementos insetos o grillas).
  - **`--brand`**: color de marca / color primario (botones primarios, enlaces destacados, acentos).
  - **`--brand-strong`**: versión más intensa del color de marca (hover/active).
  - **`--brand-soft` / `--brand-soft-strong`**: fondos/alt-accent semitransparentes para estados o resaltados.
  - **`--brand-foreground`**: color del texto sobre elementos con `--brand` (ej. texto dentro de botones primarios).
  - **`--brand-border`**: borde semitransparente relacionado con marca (bordes y outlines suaves).
  - **`--brand-glow` / `--brand-glow-strong`**: colores para sombras/efecto glow (usados por utilidades `.brand-glow*`).
  - **`--foreground`**: color principal del texto en la UI (texto estándar).
  - **`--foreground-strong`**: texto con mayor contraste (titulares).
  - **`--muted-foreground` / `--faint-foreground`**: texto de apoyo, descripciones o placeholders.
  - **`--border` / `--border-strong`**: color de bordes y líneas (inputs, tarjetas, separadores).
  - **`--input`**: fondo/borde de campos de formulario (inputs).
  - **`--ring`**: color del anillo de foco (focus outline).
  - **`--destructive` / `--destructive-foreground`**: color para acciones destructivas (eliminar, peligro) y su texto.
  - **`--chart-1` .. `--chart-5`**: paleta para gráficos/plots (cada `--chart-*` se usa como serie de color).
  - **`--radius`**: radio base para bordes; afecta `--radius-sm/md/lg/xl`.
  - **`--sidebar*`**: colores específicos para la barra lateral (fondo, texto, acentos, borde, ring).
  - **`--panel-grid`**: patrón de fondo (gradient) usado por la utilidad `.panel-grid`.

- **`@theme inline` — Tokens mapeados (`--color-*`, fuentes, radios):**
  - Crea variables tipo `--color-primary`, `--color-surface`, `--color-page`, etc., que apuntan a los `:root` anteriores.  
  - Ejemplo: `--color-primary: var(--brand)` → usar `--color-primary` en componentes hace que reciban el valor de `--brand`.
  - También define `--font-*` y `--radius-sm/md/lg/xl` calculadas desde `--radius`.
  - Cambiar en `:root` automáticamente actualiza los `--color-*` mapeados aquí.

- **`@layer base` — Aplicaciones globales:**
  - `* { @apply border-border outline-ring/50; cursor: none; }`  
    - `@apply border-border/outline-ring` usa utilidades de Tailwind (normalmente ligadas a los tokens de color/borde/ring).  
    - **`cursor: none`**: oculta el cursor en toda la página (¡cuidado; cambia mucho la interacción si lo quitas o cambias).
  - `html { @apply bg-page; }` y `body { @apply bg-page text-foreground font-sans; }`  
    - `bg-page` y `text-foreground` son utilidades (Tailwind o definidas por el proyecto) que aplican el color de fondo `--color-page` y color de texto `--color-foreground`. Cambiar `--page` o `--foreground` afectará aquí.

- **`@layer utilities` — Utilidades personalizadas:**
  - **`.panel-grid`**: usa `--panel-grid` como fondo de patrón (modifica `--panel-grid` para cambiar ese pattern).
  - **`.brand-glow` / `.brand-glow-strong`**: sombras que usan `--brand-glow` y `--brand-glow-strong`. Cambiar estas variables altera el efecto glow.

- **Selectores globales y scrollbar:**
  - `html, body, a, button, input, textarea, select { cursor: none !important; }` — refuerza ocultar cursor.
  - `::-webkit-scrollbar-track` / `::-webkit-scrollbar-thumb` usan `--surface-soft` y `--border-strong` / `--brand` al hover — cambios en esas variables actualizan el aspecto del scrollbar.

- **`@custom-variant dark (&:is(.dark *));`**
  - Define una variante personalizada para modo oscuro que apunta a la clase `.dark`. Si el proyecto activa `.dark` en `html`/`body`, se pueden usar variantes `dark:` o similares (dependiendo de la integración) para cambiar estilos.

Qué cambiar para qué efecto (resumen rápido)
- Cambia `--page` → cambia fondo global de la página.
- Cambia `--foreground` → cambia casi todo el texto primario.
- Cambia `--surface` / `--surface-foreground` → cambia tarjetas, popovers y su texto.
- Cambia `--brand` → cambia color primario (botones, enlaces, acentos).
- Cambia `--brand-glow*` → altera sombras/glow de componentes que usan `.brand-glow*`.
- Cambia `--border` / `--border-strong` → actualiza todos los bordes e inputs.
- Cambia `--destructive` → color de botones/acciones peligrosas.
- Cambia `--chart-*` → colores de series en gráficos.
- Cambia `--radius` → radio global de bordes (afecta `--radius-*`).
